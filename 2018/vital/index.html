怎么去维护一个好的博客

发表于 2018-09-13 | 分类于 生活 | | 热度: ℃

这是自己从未发现的一个bug，那就是写博客的意义，按照以前的来看，我写的应该叫作笔记，那么如何维护一个比较好的技术博客呢？怎么安排时间？怎么保持长年的稳定输出？

时刻考虑对读者的价值，而不是自己很爽

怎么写文章

写文档的时候采用top-down的思路，先讲why ,再讲how，甚至把更多的笔墨放在why 上，让读者了解背景，问题产生的原因，有哪些可能的解决方案，哪些方案较好。这么一路下来，也能帮助你自己理清思路，甚至帮你发现自己理解的漏洞。

所以如果只是平时的一些想法或者一些bug的解决应当放在笔记里，以后整理了，思考了，复盘了再采取top-down的思路写下来，在这个不断的过程中，形成自己的思维方式与写作的特色。

时间安排

想写作，一定要有固定的写作时间。而想给自己空出一个固定时间最大的问题在于你是否真的想做这件事，能重视到什么程度。

每天有大量的时间都耗在了手机上一些社交软件上，还有部分时间就是花在了游戏上，特别是周末。虽然一直都比较注重时间的安排，但总是会受到情绪的影响，什么也不想做。

长期稳定输出

no in no out
trust yourself

要想有稳定的输出，就需要稳定的输入，而且是有质量、有吸收的输入，一定要坚持思考和写笔记。
然后就是写了很多但是仍然无人问津，但是依然要相信自己，要坚持寻找自己的特色并不断完善。
最后就是共勉。
